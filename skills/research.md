# Skill: research

Deep research that pulls from several sources and saves what it finds into the wiki, so the next person
(or the next you) does not start from nothing.

Use this when the human asks you to research a topic, not just answer from memory.

## Steps

1. **Look local first.** Read `wiki/index.md`, then open the pages that touch the topic, and check `raw/`.
   You may already have part of the answer. Do not re-research what is already in the brain.
2. **Search the web (built in).** Use your own web search to fill the gaps. This needs no setup.
3. **Go deeper (optional, if set up).** If a web-search MCP server is configured (see `skills/README.md`),
   use it for broader, fresher coverage than built-in search alone. Run a few focused queries, not one.
4. **Synthesise.** Pull the threads together into a clear answer. Note where sources disagree.
5. **Save it.** Write or update a wiki page with the findings and cite your sources (local pages and web
   links). Then follow the saving rule in `CLAUDE.md`: it goes in `wiki/index.md`, gets a `log.md` line, and
   is linked from at least one existing page, not left floating alone.

The point: research compounds. A question answered today becomes a wiki page that answers it instantly
tomorrow.
